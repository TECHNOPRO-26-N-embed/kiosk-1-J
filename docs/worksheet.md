## 2. 要求仕様書（顧客ヒアリング結果）

### 2.1 顧客の要望（ヒアリングメモ）

* 書籍提出・返却・延滞管理
* ID・日付計算が多い
* 借りるときに料金がかかる
* 年中無休想定・返却は営業時間内
* C言語のみ
* ID発行は紙の想定
* 本の情報の取り込みは手入力想定→CSVで一括ならいいな
* ID登録の追加・削除できたらやってほしい
* ７泊８日から延滞料金かかる
* 登録書籍は膨大、およそ１０万冊～１００万冊くらい
* 操作のログは提案

### 2.2 機能要件

| 機能ID | 機能名 | 概要 |
| :---: | :--- | :--- |
| F01 | ユーザーID登録 | ユーザー名、貸出記録、個人情報(TEL,mail等) |
| F02 | 書籍ID登録 | 本の名前と日付 |
| F03 | 貸出確認処理 | 貸出可能かチェック、料金 |
| F04 | 返却処理 | 返却したかどうかチェック、延滞料金 |
| F05 | 貸出返却履歴保存 | 貸出・返却履歴をCSVファイルに記録 |
