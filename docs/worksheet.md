## 2. 要求仕様書（顧客ヒアリング結果）

* 借りるときに料金がかかる

* 在庫管理はあればうれしい

## 4. 詳細設計（アルゴリズム）

### 4.1 メインループ

> メニュー表示→入力受付→処理分岐→ループの流れを記述してください。
> フローチャートまたは疑似コード（箇条書き）で記述してください。

```
1.ユーザーIDログイン
　	①情報正しい：ログイン成功
	②間違い：エラーメッセージ
2.メインメニューを表示する
	①書籍貸出
	②書籍返却
	③履歴保存
	⓪ログアウト
3. ユーザーの入力を受け取る（scanf）
4. 入力に応じて分岐する
	1の場合: 貸出処理を呼び出す
	2の場合: 返却処理を呼び出す
	3の場合: 履歴表示処理を呼び出す
	0の場合: ログアウトして終了（１へ）
	それ以外: 「正しい番号を入力してください」と表示
4. メインメニューに戻る（2へ）
```

### 4.2 個別機能のアルゴリズム

> 各機能（購入処理、ログ保存、ログ表示など）の処理フローを記述してください。

#### 機能1:ユーザーIDの登録処理

```
1.ユーザー名の登録（入力）
2.ユーザーの個人情報の登録（入力）
3.ユーザー登録確認（yes/noで確認）
4.ユーザー情報確認（ユーザー名、ユーザーIDなどが見れる）
``` 

#### 機能2: 書籍ID登録

```
1.書籍のIDを入力させる
2.書籍のIDを受け取る(本の情報、タイトルなど)
3.入力されたIDが正しいか(NULL、入力できない文字)
4.書籍IDが重複してないかチェック
5.この書籍を借りていないか、借りられているかチェック(借りていた場合返却)
5.IDを登録
6.データの保存(データベースに書き込み)
7.(借りる)確認画面「この本を借りますか」(1:はい/2:いいえ)
8.(返　す)確認画面「この本を返却しますか」(1:はい/2:いいえ)
9.「はい」→「貸出処理/返却処理」へ遷移
10.「いいえ」→「キャンセルしました」→ メインメニューへ戻る
```

#### 機能3: 貸出処理画面

```
1.画面にid登録
2.本のidを入力
3.listで確認
4.time.hを使って借り出し日を決める
5.計算して返却日を見せる
6.最終確認してcsvにログ記録
```

#### 機能4: 返却処理画面
```
      1. 返却処理を選択する。
      2. ユーザーIDを入力する。
      3. 書籍IDを入力する。
      4. 貸出データを確認する。
      5. 返却期限を超えているか確認する。
      6. 延滞している場合は延滞料金を表示する。
      7. 返却情報をCSVに保存する。
      8. 「返却完了」を表示する。
      9. メインメニューに戻る。
```

#### 機能5: 貸出返却履歴保存
```
1.CSVファイルを追記モードで開く（fopen "a"）
2.貸出データを1行書き込む（fprintf）
   ①ユーザーID
   ②書籍ID
   ③書籍タイトル
   ④貸出日
   ⑤返却期限
   ⑥返却日
   ⑦延滞状況
3.ファイルを閉じる（fclose）
```
