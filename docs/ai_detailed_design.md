# 詳細設計書

## 5. 関数仕様

### 5.1 ユーザー認証・管理

#### int authenticate_user(const char* user_id);
- 役割: ユーザーIDの有効性確認（users.csvの存在確認と利用可否確認）。
- 入力:
  - user_id: 認証対象ユーザーID
- 出力: なし
- 戻り値:
  - 0: 認証成功
  - -1: user_id不正
  - -2: ユーザー未登録
  - -6: CSV読込失敗
- 処理概要:
  1. 引数チェック
  2. users.csv検索
  3. 利用可否確認

#### int load_users_from_csv(const char* csv_path);
- 役割: ユーザー情報をメモリにロード。
- 入力:
  - csv_path: users.csvのパス
- 戻り値:
  - 0: 成功
  - -6: ファイルオープン/解析失敗

### 5.2 書籍管理

#### int load_books_from_csv(const char* csv_path);
- 役割: 書籍情報をメモリにロード。
- 入力:
  - csv_path: books.csvのパス
- 戻り値:
  - 0: 成功
  - -6: ファイルオープン/解析失敗

#### int is_valid_book_id(const char* book_id);
- 役割: 書籍IDが登録済みか判定。
- 入力:
  - book_id: 対象書籍ID
- 戻り値:
  - 0: 有効
  - -1: 引数不正
  - -2: 未登録書籍ID

#### int get_book_stock(const char* book_id);
- 役割: 在庫冊数取得。
- 入力:
  - book_id: 対象書籍ID
- 戻り値:
  - 0以上: 在庫数
  - -1: 引数不正
  - -2: 書籍未登録

### 5.3 貸出・返却管理

#### int can_user_borrow_book(const char* user_id, const char* book_id);
- 役割: 貸出可否を総合判定。
- 判定条件:
  1. ユーザーIDが有効
  2. 書籍IDが有効
  3. 在庫が1冊以上
  4. 同一書籍の貸出中レコードがある場合、貸出不可
- 入力:
  - user_id: 借用ユーザーID
  - book_id: 対象書籍ID
- 戻り値:
  - 0: 貸出可
  - -1: 引数不正
  - -2: ユーザーまたは書籍未登録
  - -3: 在庫不足
  - -6: 履歴読込失敗

#### int checkout_book(const char* user_id, const char* book_id, const char* date);
- 役割: 書籍貸出処理を実行。
- 入力:
  - user_id: 借用ユーザーID
  - book_id: 対象書籍ID
  - date: 貸出日
- 戻り値:
  - 0: 成功
  - -1/-2/-3/-5/-6/-7: 失敗
- 主な副作用:
  - 支払料金を確定
  - books.csv在庫更新（-1）
  - transactions.csvへ貸出レコード追加

#### int return_book(const char* user_id, const char* book_id, const char* date);
- 役割: 書籍返却処理を実行。
- 入力:
  - user_id: 返却ユーザーID
  - book_id: 対象書籍ID
  - date: 返却日
- 戻り値:
  - 0: 成功
  - -1: 引数不正
  - -2: 未登録データ
  - -4: 借用者不一致
  - -6/-7: CSVまたは日付処理失敗
- 主な副作用:
  - 延滞料金算出
  - books.csv在庫更新（+1）
  - transactions.csv返却情報更新

#### int save_checkout_log(const char* user_id, const char* book_id, const char* date, const char* action);
- 役割: 操作ログをCSVへ記録。
- 入力:
  - user_id: 実行ユーザーID
  - book_id: 対象書籍ID
  - date: 操作日
  - action: CHECKOUT/RETURN/ERROR
- 戻り値:
  - 0: 成功
  - -6: 保存失敗

### 5.4 料金・日付計算

#### int calculate_rental_fee(const char* book_id);
- 役割: 書籍マスタから貸出料金を取得。
- 入力:
  - book_id: 対象書籍ID
- 戻り値:
  - 0以上: 料金
  - -1/-2/-6: 失敗

#### int calculate_late_fee(const char* checkout_date, const char* return_date);
- 役割: 返却遅延分の延滞料金を計算。
- 仕様:
  - 貸出日+8日以内は0円
  - 超過分に応じて延滞料金を返す（単価は設定値）
- 入力:
  - checkout_date: 貸出日
  - return_date: 返却日
- 戻り値:
  - 0以上: 延滞料金
  - -1/-7: 失敗

### 5.5 通知

#### void notify_due_date(const char* user_id, const char* book_id, const char* due_date);
- 役割: 返却期限前通知を送信。
- 仕様:
  - 返却期限1日前の対象に通知
  - 通知失敗時はログへ記録
- 入力:
  - user_id: 通知先ユーザーID
  - book_id: 対象書籍ID
  - due_date: 返却期限日

### 5.6 オプション

#### int add_user_point(const char* user_id, int points);
- 役割: ユーザーポイント加算。

#### int get_user_point(const char* user_id);
- 役割: ユーザーポイント参照。

### 5.7 表示

#### void print_error(const char* message);
- 役割: エラー表示。

#### void print_log(const char* message);
- 役割: 操作ログ表示。

## 6. 機能別処理フロー

### 6.1 ログイン
1. ユーザーID入力。
2. authenticate_user実行。
3. 成功時はメインメニュー遷移。
4. 失敗時はエラー表示し再入力。

### 6.2 書籍貸出
1. ログイン済みユーザーが書籍ID入力。
2. is_valid_book_idで登録確認（未登録ならエラー）。
3. can_user_borrow_bookで貸出可否判定。
4. calculate_rental_feeで貸出料金計算。
5. 支払い処理（成功必須）。
6. checkout_bookで貸出確定。
7. books.csv在庫更新、transactions.csvへ貸出行保存。
8. save_checkout_logで操作履歴保存。

### 6.3 書籍返却
1. ログイン済みユーザーが書籍ID入力。
2. 取引履歴から貸出中レコード検索。
3. 貸出ユーザーIDと入力ユーザーIDを照合。
4. 不一致なら返却不可エラー。
5. calculate_late_feeで延滞料金計算。
6. 必要なら延滞料金支払い処理。
7. return_bookで返却確定。
8. books.csv在庫更新、transactions.csv返却日と延滞料金更新。
9. save_checkout_logで操作履歴保存。

### 6.4 在庫照会
1. 書籍ID入力。
2. is_valid_book_idで存在確認。
3. get_book_stockで在庫取得。
4. 在庫数表示。

### 6.5 ログ参照
1. 操作ログCSVを読み込み。
2. 日付またはユーザーIDで絞り込み（任意）。
3. 一覧表示。

### 6.6 返却期限前通知
1. 定期処理（日次）で貸出中データを走査。
2. 返却期限が翌日のレコード抽出。
3. notify_due_dateを実行。
4. 通知結果をログ保存。

### 6.7 メインループ（疑似コード）
- 起動時にusers.csv、books.csv、transactions.csvを読み込む。
- ログイン成功まで以下を繰り返す。
  - ユーザーID入力を表示する。
  - ユーザーIDを入力受付する。
  - authenticate_userを実行する。
  - 認証失敗ならエラーを表示して再入力する。
- ログイン成功後、ログイン状態フラグをtrueにして以下を繰り返す。
  - メニューを表示する。
    - 1: 書籍貸出
    - 2: 書籍返却
    - 3: 在庫照会
    - 4: ログ参照
    - 0: ログアウト
  - メニュー番号を入力受付する。
  - 入力値で処理分岐する。
    - 1の場合: 書籍ID入力 -> 貸出可否判定 -> 料金計算・支払い -> 貸出確定 -> CSV保存
    - 2の場合: 書籍ID入力 -> 返却可否判定 -> 延滞料金計算・支払い -> 返却確定 -> CSV更新
    - 3の場合: 書籍ID入力 -> 在庫数取得 -> 結果表示
    - 4の場合: ログ読込 -> 一覧表示
    - 0の場合: ログイン状態フラグをfalseにしてログイン画面へ戻る
    - その他: 入力エラーを表示してメニュー再表示
- 終了条件（システム停止指示）まで、ログイン処理とメニュー処理を繰り返す。

## 7. バリデーション方針
- ユーザーID: 桁数、文字種を検証。
- 書籍ID: プレフィックスと桁数を検証。
- 日付: 妥当日付であること、返却日は貸出日以降であることを検証。
- 金額: 0以上の整数。

## 8. 性能・運用上の考慮
- 10万冊から100万冊を想定し、検索はIDキー中心で実装。
- CSV全件読み込みと差分更新を分離し、不要な再書き込みを避ける。
- ログは追記専用で運用し、日付単位でローテーション可能とする。

## 9. 未確定事項
- 延滞料金の具体単価（1日あたり固定額など）。
- 営業時間外返却受付の扱い（翌営業日扱いなど）。
- 通知チャネル（画面表示、メール、印字）。

## 10. 想定外操作時の対処
- 異常系シナリオ: ユーザーIDが空欄または形式不正：対処方法: エラーメッセージを表示し、再入力を要求する。
- 異常系シナリオ: 未登録ユーザーIDでログイン要求：対処方法: 認証失敗としてログ記録し、ログイン画面に戻す。
- 異常系シナリオ: メニューで定義外の番号を入力：対処方法: 入力エラーを表示し、同一メニューを再表示する。
- 異常系シナリオ: 書籍IDが空欄または形式不正：対処方法: エラーメッセージを表示し、書籍ID再入力に戻す。
- 異常系シナリオ: 未登録の書籍IDを入力：対処方法: 未登録エラーを表示し、処理を中断してメニューへ戻す。
- 異常系シナリオ: 在庫が0冊の書籍を貸出要求：対処方法: 貸出不可を表示し、処理を中断してメニューへ戻す。
- 異常系シナリオ: 返却時に貸出ユーザーIDと操作ユーザーIDが不一致：対処方法: 返却不可を表示し、監査ログに記録する。
- 異常系シナリオ: 日付の逆転（返却日 < 貸出日）や不正日付入力：対処方法: 日付不正エラーを表示し、日付再入力を要求する。
- 異常系シナリオ: 支払い処理が失敗または中断：対処方法: 貸出確定を行わず、失敗内容を表示して再試行または中止を選択させる。
- 異常系シナリオ: CSVファイルが存在しない/開けない：対処方法: ファイルI/Oエラーを表示し、処理を安全停止して管理者対応を促す。
- 異常系シナリオ: CSV書き込み失敗（ディスク容量不足・権限不足）：対処方法: トランザクション確定を中止し、ロールバックしてエラーログを残す。
- 異常系シナリオ: 同一書籍に対する重複貸出要求が同時発生：対処方法: 排他制御で先勝ちにし、後続要求は貸出不可として再試行を案内する。
- 異常系シナリオ: 通知送信失敗：対処方法: 失敗をログに残し、次回バッチで再送対象として扱う。
- 異常系シナリオ: 想定外例外や内部エラー発生：対処方法: 処理を中断して汎用エラーを表示し、詳細は障害ログへ出力する。
